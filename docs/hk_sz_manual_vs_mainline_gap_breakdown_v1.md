## 3. 当前根因总结

### 3.1 parser 根因

DOCX 解析后的 `ClauseUnit` 与原文行号存在错位，导致：

- `line:160-164` 的资格门槛文本在全文里存在
- 但对应的 `LegalFactCandidate` 没有稳定落到这些行
- 新链规则因此无法稳定生成 `RP-QUAL-003`

### 3.2 rule 根因

`RULE-QUAL-PERF-DUP-001` 当前可被裸 `得分` 触发，导致：

- 很多普通评分条款也被吸进 `RP-QUAL-004`
- 资格业绩重复门槛问题被噪声稀释

### 3.3 formal 根因

`RP-EVID-001` 在旧链里虽有命中，但 formal 选证时没有优先保留：

- `深圳市医疗器械检测中心`
- 这类“指定机构名称”的强证据

### 3.4 输出根因

合同与报价类问题缺少独立专项规则，导致：

- 文本虽已存在
- 但只会落入泛化合同风险
- 不利于和人工审查结果逐条对齐

## 4. 最小修复方案

### Sprint A：资格门槛簇闭环

- 为 `legal_fact_extraction` 增加 `raw_text` 行级 fallback
- 当原文行出现 `科技型中小企业 / 高新技术企业 / 纳税信用 / 成立满 / 同类项目业绩` 时，直接补生成 `LegalFactCandidate`
- 让 `RP-QUAL-003`、`RP-QUAL-004` 能稳定由新链生成实例

### Sprint B：指定机构证据闭环

- 为 `RP-EVID-001` 增加 `ReviewPointContract`
- 增加 `RULE-EVID-SOURCE-001`
- 增加 `AUTH-RP-EVID-001-001`
- formal 侧优先保留命中 `检测中心 / 检测机构 / 税务部门` 的 direct quote

### Sprint C：规则误触发收敛

- 收窄 `RULE-QUAL-PERF-DUP-001`
- 仅在出现 `同类业绩 / 类似项目业绩 / 业绩不少于` 时触发
- 不再由裸 `得分` 触发

### Sprint D：合同与报价最小专项

- 增加 `履约保证金转质量保证金或长期无息占压`
- 增加 `第三方检测费用无论结果均由中标人承担`
- 增加 `以预算金额比例设最低报价门槛`

本 Sprint 先走 `risk_rules` 直出，优先解决“能不能稳定出结果”。

## 5. 本轮不做但必须进入后续 Sprint 的项

- `GB/T 99999-2024` 标准真伪/适用性校验
- `100ms / 0.1-0.5mm / 500kg / ±10%` 技术参数归一与冲突检测
- 技术参数的“不允许正偏离”与“允许±10%偏差”约束矛盾比对
- 合同条款细粒度法理聚类，而不是仅靠关键词专项
