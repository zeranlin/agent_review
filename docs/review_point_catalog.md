# 标准审查点清单

## 说明

本文档汇总当前 `agent_review` 内置的 **75 个标准审查点**，用于：

- 快速查看当前系统覆盖范围
- 对照新增母题与真实文件回归结果
- 规划后续开发优先级

字段说明：

- `formal稳定度`
  - `是`：已在多份真实文件中较稳定进入 `formal`
  - `部分`：能识别、偶尔可 formal，但仍依赖证据锚点或文件质量
  - `否`：目前多停留在 `suspected / manual_confirmation / reviewer`
- `优先级`
  - `P1`：应优先补强，直接影响陌生文件风险点识别
  - `P2`：重要，但次于主链精度问题
  - `P3`：持续增强项
- `下一步最该补的能力`
  - `字段`：继续补结构化抽取
  - `锚点`：继续优化主证据/条款窗口
  - `要件`：继续补适法性闭合
  - `LLM二审`：继续补强度、主证据、误报压制复核

当前任务库元数据已开始沉淀到代码结构中：

- `risk_family`
  - 用于把审查点归到政策、评分、限制竞争、合同、模板、结构、人员等母题
- `target_zones`
  - 用于表达该审查点主要依赖哪些 zone 的条款
- `required_fields`
  - 用于把审查点元数据直接映射进 planning 的抽取需求

## 备注

当前最值得优先补强的方向，不是继续无限扩行业规则包，而是围绕以下 `P1 + formal稳定度=否/部分` 的审查点继续压实：

- `RP-PROC-001`
- `RP-PROC-002`
- `RP-QUAL-001`
- `RP-QUAL-002`
- `RP-SCORE-012`
- `RP-CONTRACT-011`
- `RP-PRUD-003`

这些点已经能在真实陌生文件里识别出风险方向，但距离稳定进入 `formal`，还主要差：

- 更准的主证据锚点
- 更完整的结构化字段
- 更清晰的适法性要件闭合
- 更强的 LLM 二审强度判断
