# 《agent_review 官方 19 点映射表 v1》

## 结论

这批官方 19 点不会推翻当前 `ParserEngine -> ComplianceEngine -> ReportEngine` 三引擎主链。

主要影响位于两层：

1. `ParserEngine`
需要继续补“资格门槛代理条件、检测/标准、技术参数区间、评分指标”这几类事实槽位。

2. `ComplianceEngine`
需要把官方专项规则从“泛化 review point”收成“可维护的官方规则注册表 + authority 绑定 + 报告标题映射”。

## P1 开发顺序

1. 先补 `OFFICIAL-SZDL-012` 到 `OFFICIAL-SZDL-019`
原因：这组已经有较稳定的 parser/authority 基础，最快能形成“官方口径 -> 正式报告”的闭环。

2. 再补 `OFFICIAL-SZDL-001` 到 `OFFICIAL-SZDL-005`
原因：这组需要把“显性资格门槛”升级为“法理驱动的隐性门槛识别”。

3. 最后压 `OFFICIAL-SZDL-009` 到 `OFFICIAL-SZDL-011`
原因：这组要继续把参数槽位、区间语义和偏离规则收得更细。
